=== HOW TO PUSH CHANGES ===

After editing files locally, run these commands in the
project folder (D:\assortments\web retardation):

1. See what changed:
   git status

2. Stage all changes:
   git add .

3. Commit with a message:
   git commit -m "what you changed"

4. Push to GitHub:
   git push

That's it — GitHub Pages automatically redeploys
within a minute or two.

=== CUSTOM DOMAIN (if set up) ===

A CNAME file in the repo root tells GitHub which
domain to use. Your registrar points DNS to GitHub's
Pages IPs (185.199.108.153, etc.) or CNAMEs to
nam-yan.github.io.
